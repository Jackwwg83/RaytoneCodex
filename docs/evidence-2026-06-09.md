## Implemented

- SwiftPM macOS app target: `RaytoneCodex`
- Bundle-first Codex CLI resolver with development fallback
- One app bundle staging flow in `script/build_and_run.sh`
- Codex-like app shell:
  - single left sidebar with top actions and project/thread groups
  - central transcript and bottom composer
  - right context inspector
- Session-level prompt runner: composer and menu actions call
  `SessionStore.runPrompt()`, which appends user, command, and agent transcript
  rows from a real Codex CLI result
- Session smoke automation in `script/build_and_run.sh --session-smoke`
- UI smoke automation in `script/build_and_run.sh --ui-smoke`
- Bundle audit automation in `script/build_and_run.sh --bundle-audit`
- ZIP and DMG package generation in `script/build_and_run.sh --package`
- Package artifact audit that extracts ZIP and mounts DMG before checking the
  packaged app and bundled CLI
- Package artifact SHA-256 matching: ZIP and DMG packaged app/CLI binaries must
  match the current staged bundle, so stale artifacts are rejected
- Release manifest at `Contents/Resources/RaytoneCodexRelease.json`, checked by
  bundle and package audits for app metadata, Codex reference, CLI version, and
  notice file paths
- Stage lock in `script/build_and_run.sh` so concurrent smoke/package commands
  do not delete each other's staged app or development CLI
- Empty signing baseline at `Signing/RaytoneCodex.entitlements`
- Hardened runtime signing path when a signing identity is available
- Strict release gate in `script/build_and_run.sh --release-audit`

## Not Yet Proved

This machine has no trusted code-signing identity. Therefore, the current
evidence does not prove a notarized one-install release. It proves the staged
bundle shape, the copied CLI binary, license/notice inclusion, the SwiftUI
runtime path, and a real UI window using the same staged CLI in a local
development launch.

Release proof still requires:

- Raytone Developer ID signing of `Contents/Resources/codex`
- Raytone Developer ID signing of `RaytoneCodex.app`
- hardened runtime on both signed executables
- notarization and stapling
- clean-machine Gatekeeper validation
